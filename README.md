MetaTk is a slopware, but just like most my slopware, it's *useful*. If the Facebook's brightest --- at the risk of making this rather topical -- are "drafted" into solving challenges for Meta's Slop Machine through, then, I, a sole person who has lost his access to Codex and has to use GPT 5.5 as agent, must be excused for utilizing Agentic Slop Machines to build libraries that I will, myself, find useful in a rainy day.

MetaTk, or "Meta Programming Toolkit", is a collection of libraries and applications that adhere to principles of Software Language Engineering's definition of metaprogramming. That is, DSLs (whether native or textual) aimed at the 'meta' level of the program's semantics.

For now, MetaTk has three substrates:
- DSLtk: a header-only C++ library for authoring anything form of C++ native DSLs. It currently lacks CTRE, but it has all the rest --- static ASTs will be added too. A rewrite by the Slop Apparatus will add more instances of `constexpr`;
- Parzek: a CLI application, which uses DSLtk's combinatory parser to create a PEG-based parser generator. Yes, maybe DSLtk's odd to have a combinatory parser, but whatever;
- EkippX: the grandson of [Ekipp](https://github.com/Chubek/Ekipp), which I wrote by hand, like an ape, and the son of (Ekipp-Library)[https://github.com/Chubek/Ekipp-Library], which hasn't fallen off the apple tree too far --- but just like my late dad and I, they don't see eye-to-eye in many issues. For example, it uses DSLtk to offer its services, not as an API, but also, as a native C++ DSL;

All three libraries have an `examples` and a `manual` directory.
